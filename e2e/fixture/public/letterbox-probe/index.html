Letterbox Probe 허브

iOS 웹 클립(홈 화면 추가)에서 letterbox 동작을 meta/manifest 조합별로 비교하는 진단용 페이지 모음입니다.
각 변형을 별도 웹 클립으로 설치해 동일 기기에서 나란히 비교하세요.

기존 변형 (기준선)

  • default 기존
    viewport-fit=cover + status-bar-style=default — 기본 동작 기준선.
  • black-translucent 기존
    viewport-fit=cover + black-translucent — ICB가 status bar까지 확장되는 조합.
  • min-height hack 기존
    html을 safe-area-inset-top만큼 늘려 letterbox 영역을 문서 콘텐츠로 채우는 시도.

신규 변형

  • fullscreen manifest 신규
    manifest display: "fullscreen" + translucent meta — iOS가 fullscreen을 standalone으로 fallback하는지, 아니면 다른 지오메트리를 주는지 판별. display-mode: fullscreen / standalone 양쪽 matchMedia 표시.
  • pxfix (#527 단독 재현) 신규
    root에 height: screen.height px 명시 보정(#527 메커니즘)을 단독으로 재현. ON/OFF 토글로 보정 전후 비교 — 파란 문서 하단 밴드가 물리 바닥에 닿으면 보정 성립.
  • viewport-fit=contain 신규
    viewport-fit=contain + black-translucent — cover 제거 효과 분리. safe-area-inset-* 값이 0으로 보고되는지, ICB 높이가 달라지는지 확인.
iOS 웹 클립 설치 주의 iOS 웹 클립은 설치 시점의 meta와 manifest를 동결합니다. 변형마다 별도 웹 클립으로 설치하고, 페이지를 수정한 뒤에는 반드시 삭제 후 재추가해야 변경이 반영됩니다.
  1. Safari에서 각 변형 페이지를 직접 열기.
  2. 공유 메뉴 → "홈 화면에 추가".
  3. 변형마다 이름을 구분해 별도 아이콘으로 저장.
  4. 페이지 내용이 바뀌면 홈 화면 아이콘을 삭제하고 3단계부터 다시 진행.
